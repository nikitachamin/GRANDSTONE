# Правило именования компонентов

## Описание
Для именования компонентов Vue.js рекомендуется использовать стиль CamelCase. Это означает, что каждое новое слово в имени компонента начинается с заглавной буквы, а все остальные буквы — строчные. Такой подход помогает улучшить читаемость и поддерживаемость кода.

## Примеры
- **Правильные примеры**:
  - `AppHeader`
  - `UserProfile`
  - `ShoppingCartItem`

- **Неправильные примеры**:
  - `mycomponent`
  - `user_profile`
  - `shopping-cart-item`

## Рекомендации
1. **Согласованность**: 
   - Используйте CamelCase для всех компонентов в проекте, чтобы обеспечить единообразие. Это поможет другим разработчикам быстрее ориентироваться в коде.

2. **Суффиксы**: 
   - При необходимости добавляйте суффиксы для указания типа компонента. Это может помочь в понимании назначения компонента на первый взгляд. Примеры суффиксов:
     - `AppButton` — для кнопок приложения
     - `UserCard` — для карточек пользователей
     - `ModalDialog` — для модальных окон

3. **Читаемость**: 
   - Выбирайте имена, которые четко отражают функциональность компонента. Это улучшает читаемость кода и упрощает его поддержку. Например:
     - Вместо `DataFetcher` можно использовать `UserDataFetcher`, чтобы уточнить, какие данные компонент загружает.

4. **Избегайте аббревиатур**: 
   - Если возможно, избегайте использования аббревиатур в именах компонентов, так как это может затруднить понимание их назначения. Например, вместо `Btn` используйте `Button`.

5. **Используйте префиксы для группировки**: 
   - Если в проекте есть несколько связанных компонентов, рассмотрите возможность использования префиксов для их группировки. Например:
     - `AuthLogin` и `AuthRegister` для компонентов, связанных с аутентификацией.
